# Memory Manager

CEO 的记忆引擎。负责将对话中的重要信息写入持久化文件。

## 触发条件

当发生以下情况时，调用 `write` 工具更新记忆文件：

1. **重大决策** — 架构变更、策略决定、重要配置修改
2. **新认知** — 学习了新知识、新工具、新流程
3. **错误修复** — 发现并修复了问题，记录防止重复
4. **定时触发** — 每日日终从 `memory/YYYY-MM-DD.md` 提炼到 `MEMORY.md`

## 记忆文件层级

| 文件 | 内容 | 更新频率 |
|------|------|---------|
| `MEMORY.md` | 精选长时记忆，精华提炼 | 周审查+更新 |
| `memory/YYYY-MM-DD.md` | 当日原始日志 | 实时写入 |
| `knowledge/system/` | 结构化知识文档 | 按需更新 |

## 写入规范

### MEMORY.md 更新
- 先 `read` 现有内容
- 只追加**已验证、已决策**的内容
- 删除过时信息（注明原因）
- 每条记忆注明日期和来源

### 日记忆文件
```markdown
# 2026-06-27

## 事件
- [时间] 事件描述

## 决定
- 决策内容及理由

## 教训
- 错误及修复方式
```

## 禁止

- ❌ 不确定的信息不写入 MEMORY.md（留在日记忆中）
- ❌ 不写入猜测性内容
- ❌ 不写入用户私人信息
