# Project Management

> Skill: project-management | v1.0.0 | for: pm
> 适用场景：项目启动、任务分解、进度跟踪、风险管理、项目复盘

## 描述

对项目或复杂任务进行管理：制定计划、分解任务、跟踪进度、管理风险，输出可执行的项目文档。
用于 COO 域的项目统筹，确保任务按时按质完成。

## 前置条件

- 了解 OpenClaw 生态的 Agent 分布（参考 `knowledge/system/agents.md`）
- 了解 Agent 调度方式（sessions_spawn / sessions_send）

## 输入

| 字段 | 类型 | 说明 |
|------|------|------|
| project_name | string | 项目名称 |
| goal | string | 项目目标（要达成什么）|
| deadline | string | 截止日期 |
| constraint | string | 约束条件（可选）|

## 使用方法

### 触发方式

在 pm session 中调用：

```
启动项目：{项目名称}
目标：{目标描述}
截止：{YYYY-MM-DD}
约束：{如有限制}
```

## 输出格式

```markdown
# 项目管理文档：{项目名称}

> 项目状态：🟡 进行中
> 创建日期：{YYYY-MM-DD}
> 截止日期：{YYYY-MM-DD}

## 1. 项目目标
<!-- 可量化的目标描述 -->

## 2. 任务分解（WBS）

| 任务ID | 任务名称 | 负责人 | 截止日期 | 状态 |
|--------|---------|--------|---------|------|
| T-01 | ... | ... | YYYY-MM-DD | ✅完成/🟡进行/⬜待启动 |
| T-02 | ... | ... | YYYY-MM-DD | ✅完成/🟡进行/⬜待启动 |

## 3. 里程碑

| 日期 | 里程碑 | 交付物 | 状态 |
|------|--------|--------|------|
| YYYY-MM-DD | ... | ... | ✅/🟡/⬜ |

## 4. 风险登记

| 风险 | 可能性 | 影响 | 应对 |
|------|--------|------|------|
| ... | 高/中/低 | 高/中/低 | ... |

## 5. 进度跟踪

| 周次 | 计划进度 | 实际进度 | 偏差 | 调整 |
|------|---------|---------|------|------|
| W1 | 20% | X% | +Y%/-Y% | ... |
```

## 限制

- 不做具体执行（那是执行层 Agent 的职责）
- 不决定资源分配（那是 COO 的职责）
- 项目管理文档仅提供框架，实际执行由对应 Agent 负责

## 示例

### 示例：项目启动

**输入：**
```
启动项目：OpenClaw 全生态标准化 v2.0
目标：完成所有 SOP、工作流、技能标准化
截止：2026-06-30
约束：白天不打断日常工作
```

**输出：**
```
# 项目管理文档：OpenClaw 全生态标准化 v2.0

## 1. 项目目标
- 完成全生态 SOP 体系建设（≥20 个 SOP）
- 完成 CTO/COO/CA 域 Skills 部署（≥10 个 Skill）
- 56/56 文件达到 100% 合规率

## 2. 任务分解
| 任务ID | 任务名称 | 负责人 | 截止 | 状态 |
|--------|---------|--------|------|------|
| T-01 | 自进化体系 SOP | CEO | 2026-06-27 | ✅完成 |
| T-02 | CTO 域 Skills | CTO | 2026-06-28 | 🟡进行 |
| T-03 | COO 域 Skills | COO | 2026-06-28 | ⬜待启动 |
| T-04 | Pending SOPs | CEO | 2026-06-29 | ⬜待启动 |

## 3. 风险登记
| 风险 | 可能性 | 影响 | 应对 |
|------|--------|------|------|
| 工作量超负荷 | 高 | 高 | 优先级排序，优先完成高优先级项 |
```
