# Hardware Ops

> Skill: hardware-ops | v1.0.0 | for: he
> 适用场景：硬件故障诊断、设备状态评估、运维建议

## 描述

对硬件相关问题进行诊断和评估，识别硬件层面的问题，提供运维建议。
用于 CTO 域的硬件支持。

## 前置条件

- 了解 OpenClaw 部署环境（Windows WSL2 / Linux / Docker）
- 能读取系统资源数据（CPU / 内存 / 磁盘）

## 输入

| 字段 | 类型 | 说明 |
|------|------|------|
| issue | string | 问题描述或症状 |
| system_info | string | 系统信息（可选）|

## 使用方法

```
请诊断以下硬件/系统问题：
{问题描述}
```

## 输出格式

```markdown
# 硬件运维诊断报告

## 问题概述
{描述}

## 诊断结果
| 检查项 | 结果 | 说明 |
|--------|------|------|
| CPU | 正常/异常 | |
| 内存 | 正常/异常 | |
| 磁盘 | 正常/异常 | |

## 问题定位
{分析}

## 运维建议
- [ ] ...
```

## 示例

**输入：**
```
WSL2 内存使用率持续在 90% 以上，经常崩溃
```

**输出：**
```
# 硬件运维诊断报告

## 问题概述
WSL2 内存使用率持续 90% 以上

## 诊断结果
| 检查项 | 结果 | 说明 |
|--------|------|------|
| WSL2 内存配置 | ⚠️ 异常 | 默认无内存上限，共享 Windows 内存 |
| 磁盘空间 | ✅ 正常 | 剩余 50GB |
| CPU | ✅ 正常 | 使用率波动在 30-60% |

## 问题定位
WSL2 默认配置下，内存使用无上限，与 Windows 共享内存资源。
90% 使用率可能是 WSL 内部进程（node）累积导致。

## 运维建议
1. 设置 WSL 内存上限（.wslconfig: memory=4GB）
2. 定期重启 WSL 释放内存
3. 监控 node 进程数量
```
